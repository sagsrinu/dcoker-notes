**Docker Lab - Structured Notes**

---

**Phase 1: Setup & Verification**

1. Install Docker
`sudo yum install docker -y`

2. Start Docker daemon
`sudo systemctl start docker`

3. Check Docker service status
`systemctl status docker`

4. Check version (short)
`docker --version`

5. Check version (full client + server details)
`docker version`

---

**Phase 2: docker version output explained**

- Client and Server both show version 25.0.14
- API version: 1.44 (minimum supported: 1.24)
- OS/Arch: linux/amd64
- containerd version: 2.2.1
- runc version: 1.3.4
- docker-init version: 0.19.0

Note: `docker --version` shows only client version. `docker version` confirms both client and daemon are reachable.

---

**Phase 3: Working with Images**

6. Pull latest Ubuntu image
`docker pull ubuntu`

7. Pull a specific version
`docker pull ubuntu:20.04`
Always mention the tag if you need a specific version.

8. List all downloaded images
`docker images`

Sample output:
```
REPOSITORY   TAG       IMAGE ID       CREATED       SIZE
ubuntu       latest    0b1ebe5dd426   12 days ago   78.1MB
```

Note: Docker Hub (hub.docker.com) is the central registry where all official and community images are hosted.

---

**Phase 4: Running Containers**

9. Run container interactively
`docker run -it ubuntu /bin/bash`
Attaches terminal so you can type commands inside the container.

10. Run container in background (detached)
`docker run -dt ubuntu /bin/bash`
Container runs without attaching to your terminal.

Flag reference:
- `-i` → interactive, keeps STDIN open
- `-t` → allocates a terminal (TTY)
- `-d` → detached, runs in background
- `/bin/bash` → process to start inside container

---

**Phase 5: Inspecting Container Processes**

11. View processes running inside container
`ps -aux | grep -v grep`

This shows /bin/bash as the only active process, confirming the container runs only what it needs.
